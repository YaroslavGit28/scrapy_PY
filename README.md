# Техническое задание на проект "Демонстрационный стенд Scrapy"

## 1. Общая информация

**Название проекта:** Демонстрационный стенд Scrapy

**Цель проекта:** Показать на понятных примерах, как работает библиотека Scrapy для парсинга сайтов. Не просто "привет мир", а реальный пайплайн — от запроса до сохранённых данных.

## 2. Описание задачи (что нужно сделать)

Нужно написать несколько демонстрационных скриптов, которые показывают ключевые фишки Scrapy:

- как создать запрос и получить ответ
- как вытащить данные из HTML через CSS и XPath
- как переходить по ссылкам внутри сайта
- как структурировать данные через Items
- как обрабатывать данные через Pipeline
- как сохранять результат в JSON или CSV

Всё должно быть разложено по отдельным файлам, чтобы не смешивать разные темы в кучу. И должен быть один файл-запускалка, который прогоняет всё по очереди.

## 3. Функциональные требования (что система должна уметь)

1. Демо 1 — показать базовые сущности Scrapy (Request, Response, TextResponse).
2. Демо 2 — показать работу селекторов: как находить элементы через CSS и XPath.
3. Демо 3 — показать полный цикл паука: `start_requests`, парсинг страницы, переход по ссылкам (`follow`).
4. Демо 4 — показать работу с Items и Pipeline: сбор данных в структуру и их постобработка.
5. Демо 5 — показать настройки Scrapy и экспорт данных в файл.
6. Каждое демо должно выводить результат в консоль с пояснениями, что сейчас происходит.
7. Все демки должны запускаться одной командой из `run_all.py`.
8. Демки не должны требовать доступа к реальному интернету — либо использовать тестовые HTML-страницы, либо мокать ответы.

## 4. Нефункциональные требования (ограничения)

1. Язык программирования: Python 3.10+
2. Используемые библиотеки: `scrapy`, остальное из стандартной библиотеки
3. Интерфейс: консольный вывод
4. Архитектура: каждое демо — отдельный файл, плюс общий запускалка
5. Код должен быть читаемым, с комментариями
6. Проект должен запускаться на Windows, Linux и macOS

## Описание модулей

### 1. `demo_01_basics.py` — что такое Request и Response

Показывает самые основы Scrapy. Как создать запрос и как выглядит ответ.

```python
from scrapy.http import Request, Response, HtmlResponse

# Создаём запрос
req = Request("https://example.com")
print(f"Request: {req.url}, метод: {req.method}")

# Имитируем ответ
html = "<html><body><h1>Заголовок</h1></body></html>"
resp = HtmlResponse(url="https://example.com", body=html, encoding='utf-8')

print(f"Ответ: {resp.url}, статус: {resp.status}")
print(f"Заголовок из HTML: {resp.css('h1::text').get()}")
```

Зачем это нужно: чтобы понять, что Scrapy внутри работает с этими объектами. Ты не просто просишь "дай страницу", а работаешь с полноценными объектами.

### 2. `demo_02_selectors.py` — как искать элементы

Показывает разницу между CSS-селекторами и XPath. И как достать текст или атрибут.

```python
html = """
<div class="products">
    <div class="item" data-id="1">Товар 1</div>
    <div class="item" data-id="2">Товар 2</div>
</div>
"""

# CSS-селектор
items_css = response.css('div.item')
print(f"CSS нашёл {len(items_css)} элементов")
for item in items_css:
    print(f"  - {item.css('::text').get()}, id={item.css('::attr(data-id)').get()}")

# XPath
items_xpath = response.xpath('//div[@class="item"]')
print(f"XPath нашёл {len(items_xpath)} элементов")
```

Разница между CSS и XPath:
- CSS короче и привычнее, если знаешь вёрстку
- XPath мощнее: можно искать текст, идти вверх по дереву, искать по содержанию

### 3. `demo_03_spider_flow.py` — как паук ходит по сайту

Показывает главную логику Scrapy-паука:
1. `start_requests` — с чего начинаем
2. `parse` — разбираем страницу
3. `follow` — переходим по найденным ссылкам

```python
class SimpleSpider(scrapy.Spider):
    name = "demo_spider"
    
    def start_requests(self):
        # Начинаем с главной страницы
        yield scrapy.Request("https://example.com", callback=self.parse)
    
    def parse(self, response):
        # Парсим текущую страницу
        for link in response.css('a::attr(href)').getall():
            # Переходим по каждой ссылке
            yield response.follow(link, callback=self.parse_page)
    
    def parse_page(self, response):
        # Парсим страницу, на которую перешли
        yield {"url": response.url, "title": response.css('title::text').get()}
```

Это сердце Scrapy. Вся магия в том, что `follow` сам понимает, полный это URL или относительный, и не надо склеивать руками.

### 4. `demo_04_items_pipelines.py` — как структурировать и чистить данные

**Items** — это как схема данных. Говоришь Scrapy: "Я жду поля `title`, `price`, `url`". Pipeline обрабатывает эти данные до того, как они куда-то сохранятся.

```python
# items.py
class ProductItem(scrapy.Item):
    title = scrapy.Field()
    price = scrapy.Field()
    url = scrapy.Field()

# pipeline.py
class CleanPricePipeline:
    def process_item(self, item, spider):
        if item.get('price'):
            # Убираем знак доллара и пробелы
            item['price'] = item['price'].replace('$', '').strip()
        return item
```

Зачем это:
- Item проверяет, что ты не забыл какое-то поле
- Pipeline может чистить данные, менять формат, отбрасывать мусор

### 5. `demo_05_settings_export.py` — настройки и сохранение

Показывает, как настроить поведение паука и куда сохранять результат.

```python
settings = {
    "USER_AGENT": "Mozilla/5.0 (My Scraper)",
    "DOWNLOAD_DELAY": 1,  # ждать секунду между запросами
    "FEEDS": {
        "output.json": {"format": "json", "encoding": "utf8"},
        "output.csv": {"format": "csv"},
    }
}
```

Настройки:
- `DOWNLOAD_DELAY` — вежливый парсинг, не долбим сервер
- `USER_AGENT` — представляемся браузером, а не скриптом
- `FEEDS` — куда сохранять результат (JSON, CSV, XML)

## Какие трудности были

**1. Без интернета демки не работают**
Scrapy привык ходить в сеть. Пришлось мокать ответы — вместо реального запроса подсовывать готовый HTML.

**2. Асинхронность**
Scrapy асинхронный внутри, но для демок это неважно. Главное — показать логику, а не гнаться за скоростью.

**3. Много boilerplate-кода**
Обычный проект Scrapy требует команд `scrapy startproject`, кучи файлов. В демках я вырезал лишнее, оставил только суть.

## Какие ещё темы можно добавить

- Middlewares (как подменить User-Agent или прокси)
- Правила (`Rule` + `LinkExtractor`) для автоматического обхода
- Пауки-пауки: CrawlSpider, XMLFeedSpider, CSVFeedSpider
- Работа с JavaScript-сайтами (через Splash или Selenium, но это уже не чистый Scrapy)

Но для первого знакомства и этих пяти демок хватит. Они покрывают 80% того, что делают с Scrapy в реальных проектах.
