
# Техническое задание на проект «Демонстрационный стенд Scrapy»

## 1. Введение (или «Почему я решил это сделать»)

Когда мы впервые попробовал разобраться в Scrapy, я чуть не бросил эту затею. Официальный туториал предлагал создать проект с кучей файлов, понять, что такое `items.py`, `pipelines.py`, `settings.py`, и только потом написать своего первого паука. Это как учиться водить машину, начиная с разбора коробки передач.

Мы решили сделать иначе. Я написал несколько отдельных скриптов, каждый из которых показывает **одну** ключевую идею Scrapy. Без лишней структуры, без boilerplate-кода. Просто открываешь файл, запускаешь и видишь, как это работает. Этот проект — моя попытка построить мостик для тех, кто идёт за мной.

## 2. Цель работы

Создать набор демонстрационных скриптов, которые наглядно и пошагово показывают основные возможности фреймворка Scrapy: от базовых объектов Request/Response до полноценного паука с Items, Pipelines и экспортом в файл.

**Главная цель — показать, что Scrapy — это не страшно, а очень даже удобно, и начать его использовать может каждый, кто знает основы Python.**

## 3. Функциональные требования

Должно быть **5 демонстрационных скриптов**, каждый на отдельную тему:

### Демо 1: Базовые сущности (`demo_01_basics.py`)
- Показать создание объекта `Request`
- Показать создание объекта `Response` (или `HtmlResponse`)
- Продемонстрировать основные свойства: `url`, `status`, `body`, `css`, `xpath`

### Демо 2: Селекторы (`demo_02_selectors.py`)
- Показать поиск элементов через `response.css()`
- Показать поиск через `response.xpath()`
- Извлечение текста: `.css('::text').get()`
- Извлечение атрибутов: `.css('::attr(href)').get()`
- Продемонстрировать разницу между `.get()` и `.getall()`

### Демо 3: Полный цикл паука (`demo_03_spider_flow.py`)
- Создать класс паука, наследующий `scrapy.Spider`
- Реализовать `start_requests` (начальные запросы)
- Реализовать `parse` (обработка ответа)
- Показать переход по ссылкам через `response.follow()`
- Показать передачу данных через `yield` (Item или словарь)

### Демо 4: Items и Pipelines (`demo_04_items_pipelines.py`)
- Определить класс Item с полями (например, `title`, `price`, `url`)
- Создать простой Pipeline (например, очистка пробелов, удаление $ из цены)
- Зарегистрировать Pipeline в настройках (внутри скрипта)
- Показать, как Item проходит через Pipeline

### Демо 5: Настройки и экспорт (`demo_05_settings_export.py`)
- Показать, как задать `USER_AGENT`
- Показать, как задать `DOWNLOAD_DELAY`
- Настроить экспорт в JSON файл через `FEEDS`
- (Опционально) Показать экспорт в CSV

### Общие требования ко всем демкам:
- Каждый скрипт имеет функцию `main()`, которую можно вызвать
- Все демки не требуют доступа к реальному интернету (используют мок-данные)
- Каждый пример сопровождается пояснительным выводом в консоль

### Требования к запускалке (`run_all.py`):
- Последовательно вызывает `main()` каждой демки
- Выводит разделители между демками для читаемости
- Обрабатывает возможные ошибки импорта

## 4. Нефункциональные требования

1.  **Язык и библиотеки:** Python 3.10+. Основная библиотека — `scrapy`. Можно использовать `dataclasses` из стандартной библиотеки, но не обязательно.
2.  **Кросс-платформенность:** Скрипты должны работать на Windows, Linux и macOS. Scrapy везде работает, но на Windows могут быть нюансы с установкой (нужен Visual C++ Build Tools).
3.  **Читаемость кода:**
    - Минимум магии: никаких метаклассов и сложных декораторов
    - Комментарии, поясняющие неочевидные моменты
    - Переменные названы по-человечески
4.  **Понятность вывода:** Каждый пример выводит:
    - Заголовок с темой
    - Исходные данные (HTML, URL)
    - Результат преобразования
    - Краткое пояснение, зачем это нужно в реальной жизни
5.  **Документированность:** `README.md` + папка `wiki/` с подробным объяснением кода

## 6. Критерии приёмки

Работа считается выполненной, если:

1.  Установлен Scrapy (`pip install scrapy`).
2.  Команда `python src/run_all.py` запускает все пять демок без ошибок.
3.  Демо 1 явно показывает создание Request и Response.
4.  Демо 2 показывает поиск через CSS и XPath, извлечение текста и атрибутов.
5.  Демо 3 показывает старт паука, парсинг страницы и переход по ссылке.
6.  Демо 4 показывает определение Item и работу Pipeline.
7.  Демо 5 показывает экспорт данных в файл (хотя бы JSON).
8.  Ни одна демка не делает реальный HTTP-запрос (все данные — мокнутые).
9.  Репозиторий содержит `README.md` и папку `wiki` с документацией.
10. Последний коммит сделан не позднее установленного срока (1 неделя).
