Техническое задание на проект «Консольный файловый менеджер на os и os.system»
1. Введение (или «Почему мы решил это сделать»)
Нам нужно было придумать проект для курса по Python, который бы показал, что я умею работать с модулями os, sys и pathlib. Просто прочитать про них в документации — скучно. Гораздо лучше — написать что-то полезное, что я сам мог бы использовать. Так родилась идея сделать консольный файловый менеджер. Он не будет красивым, но будет полезным и, самое главное, покажет, как Python общается с операционной системой.

2. Цель работы
Разработать консольное приложение на Python 3.10+, которое позволяет пользователю выполнять базовые операции с файлами и папками (навигация, создание, удаление, просмотр), а также запускать системные приложения (файловый менеджер). Программа должна быть кроссплатформенной (работать на Windows, Linux и macOS) и устойчивой к ошибкам пользователя.

Главная цель проекта — учебная: на живом примере изучить и продемонстрировать работу с модулями os, os.system, sys и pathlib из стандартной библиотеки Python.

3. Функциональные требования
Программа при запуске должна показывать интерактивное текстовое меню. Пользователь вводит номер команды, и программа выполняет соответствующее действие.

Список команд:

Показать текущую папку — вывести на экран абсолютный путь к текущей рабочей директории.

Показать содержимое папки — вывести список всех файлов и подпапок внутри текущей директории. Файлы и папки должны визуально отличаться (например, разными иконками или символами).

Создать папку — запросить у пользователя имя новой папки и создать её в текущей директории. Обработать ситуацию, когда папка уже существует.

Удалить папку — запросить имя папки и удалить её. Важно: удалять можно только пустые папки. Если папка не пуста или не существует, программа должна вывести понятное сообщение об ошибке.

Удалить файл — запросить имя файла и удалить его. Проверить, что файл существует и что это действительно файл, а не папка.

Открыть файловый менеджер системы — запустить стандартное приложение для просмотра файлов (Проводник на Windows, Finder на macOS, Nautilus/Dolphin/etc. на Linux), открыв в нём текущую папку программы.

Выход — завершить работу программы.

Дополнительные функциональные требования:

После выполнения любой команды (кроме выхода) программа должна ждать нажатия клавиши Enter, чтобы пользователь успел прочитать результат, а затем очистить экран и снова показать меню.

При вводе некорректного пункта меню (не число или число вне диапазона 0-6) программа должна вывести сообщение об ошибке и показать меню снова, не завершая работу.

4. Нефункциональные требования
Кроссплатформенность: Код должен работать на Windows, Linux и macOS. Для этого необходимо определять операционную систему с помощью sys.platform и использовать соответствующие системные команды.

Язык и библиотеки: Python 3.10+. Разрешены только модули из стандартной библиотеки: os, sys, pathlib. Использование shutil или других сторонних библиотек запрещено.

Читаемость кода:

Код должен быть разбит на логические функции (например, create_folder, delete_file, print_menu).

У каждой функции должно быть понятное имя и, по возможности, docstring или комментарий.

Переменные названы по-человечески, на английском языке.

Отсутствуют «магические числа» и глубокие вложенности.

Устойчивость к ошибкам:

Программа не должна завершаться аварийно (падать с traceback) при любых действиях пользователя: ввод букв вместо цифр, попытка удалить несуществующий файл, создание папки с недопустимым именем и т.д.

Все потенциально опасные операции (удаление, создание) должны быть обёрнуты в try...except с обработкой специфических исключений (FileNotFoundError, OSError и т.д.).

Документированность: Проект должен содержать файл README.md с описанием и инструкцией по запуску, а также подробную документацию в папке wiki/, где объясняется, как работает программа и почему были выбраны те или иные подходы.

5. Состав проекта (результат работы)
После завершения работы репозиторий GitHub должен содержать:

file_manager.py — исполняемый файл с программой.

README.md — краткое описание, инструкция по установке и запуску, примеры работы.

wiki/ — папка с подробной инженерной документацией (не менее 5 файлов .md), раскрывающей устройство программы и используемые модули.

6. Критерии приёмки
Работа будет считаться выполненной и зачтенной, если:

Программа запускается без ошибок на Windows 10/11 и Ubuntu 22.04+.

Все пункты меню (1-6 и 0) работают в соответствии с функциональными требованиями.

При попытке удалить непустую папку программа выдаёт сообщение «Папка не пуста» и не удаляет её.

При вводе буквы «ы» в меню программа не падает, а пишет «Неверный выбор» и показывает меню снова.

Код написан с использованием pathlib.Path для всех операций с путями (а не os.path.join).

В коде есть определение ОС и используется правильная команда для очистки экрана (cls или clear).

Репозиторий содержит файл README.md и папку wiki с документацией.

Последний коммит в репозиторий сделан не позднее установленного срока (1 неделя с момента выдачи задания).

7. План выполнения (для себя)
День 1: Изучить документацию по pathlib, os, sys. Набросать структуру file_manager.py и основное меню.

День 2: Реализовать функции show_current_dir, list_directory, create_folder. Протестировать на Windows.

День 3: Реализовать функции delete_folder, delete_file. Добавить обработку ошибок. Протестировать на Linux в WSL.

День 4: Реализовать функцию open_file_manager и определение ОС. Добавить очистку экрана и паузу после действий.

День 5: Написать README.md и начать писать документацию в wiki/. Провести финальное тестирование на всех ОС.

День 6: Закончить документацию, исправить найденные баги.

День 7: Сделать финальный коммит, сдать проект.
