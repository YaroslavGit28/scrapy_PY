Обзор программы файлового менеджера (file_manager.py)
Эта документация написана так, чтобы в ней разобрался человек, который только начинает знакомство с Python, но хочет понять, как программы взаимодействуют с операционной системой. Задача текста — объяснить не только «что делает код», но и зачем это сделано именно так, и с какими ограничениями мы столкнулись на практике.

1. Что вообще делает программа file_manager.py
file_manager.py — это консольный файловый менеджер. Он не имеет красивого графического интерфейса (как «Проводник» в Windows), а работает через текстовое меню в терминале. Вы запускаете скрипт, видите список команд, вводите номер — и программа выполняет действие: показывает текущую папку, создаёт директорию или открывает системный проводник.

Главная цель этой программы — на живом примере показать, как Python с помощью стандартных библиотек (os, pathlib, sys) может напрямую взаимодействовать с файловой системой и даже запускать другие программы на вашем компьютере.

Вот как выглядит типичный запуск и работа:

bash
python file_manager.py

=== Файловый менеджер ===
1. Показать текущую папку
2. Показать содержимое папки
3. Создать папку
4. Удалить папку
5. Удалить файл
6. Открыть файловый менеджер системы
0. Выход

Выберите действие (0-6): 1

Текущая папка: C:\Users\Windows\Desktop\os.sys
Нажмите Enter, чтобы продолжить...
Эта программа — настоящий «швейцарский нож» для новичка: она умеет показывать, где ты находишься, что лежит в папке, создавать и удалять папки и файлы, и даже запускать внешние приложения.

2. Общая архитектура программы (из каких блоков она состоит)
Внутри file_manager.py код разбит на логические части, каждая из которых отвечает за свою задачу. Такой подход называется «разделение ответственности», и он очень важен даже в маленьких скриптах.

Блок определения ОС (в самом начале):
Здесь программа «осматривается» и решает, под какой операционной системой она работает (Windows, macOS или Linux). От этого зависят команды для очистки экрана (cls или clear) и запуска файлового менеджера (explorer ., open . или xdg-open .).

Функции-действия (хелперы):
Это отдельные функции под каждую команду меню: show_current_dir(), list_directory(), create_folder(), delete_folder(), delete_file(), open_file_manager(). Каждая делает только одно дело и ничего больше.

Функции управления интерфейсом:
print_menu() — просто выводит текст меню на экран. get_user_choice() — спрашивает у пользователя номер команды и обрабатывает ошибки (например, если ввели букву вместо цифры).

Главный цикл программы (main()):
Оно работает бесконечно (while True), пока пользователь не выберет пункт «0. Выход». На каждой итерации цикл показывает меню, получает выбор пользователя и вызывает нужную функцию-действие.

3. Блок определения ОС — почему это важно и как работает
Самое интересное и «низкоуровневое» место в программе — это самое начало. Посмотрите на этот фрагмент:

python
import sys

if sys.platform == "win32":
    CLEAR_CMD = "cls"
    FILE_MANAGER_CMD = "explorer ."
elif sys.platform == "darwin":
    CLEAR_CMD = "clear"
    FILE_MANAGER_CMD = "open ."
else:  # Linux и другие Unix-подобные
    CLEAR_CMD = "clear"
    FILE_MANAGER_CMD = "xdg-open ."
Зачем это нужно? Дело в том, что разные операционные системы используют разные команды для одних и тех же действий. На Windows команда для очистки терминала — cls, а на Linux и macOS — clear. Файловый менеджер на Windows запускается командой explorer ., на macOS — open ., а на Linux — xdg-open .. Если этого не учесть, программа будет работать только на одной ОС и сломается на всех остальных.

Как это работает? Модуль sys предоставляет доступ к системным параметрам. Атрибут sys.platform — это строка, которая идентифицирует ОС. win32 (даже на 64-битной Windows), darwin (macOS) — это стандартные значения.

4. Функции работы с файловой системой (через pathlib)
Самые важные действия — создание, удаление, просмотр — используют современный модуль pathlib. Разберем пару примеров.

Функция list_directory() — посмотреть содержимое папки
python
from pathlib import Path

def list_directory():
    current = Path.cwd()
    print(f"\nСодержимое папки {current}:\n")
    for item in current.iterdir():
        if item.is_dir():
            print(f"{item.name}")  # папка
        else:
            print(f"{item.name}")  # файл
Path.cwd() — получает текущую рабочую директорию (current working directory). Это аналог команды pwd в Linux или cd без параметров в Windows.

current.iterdir() — возвращает итератор по всем объектам (файлам и папкам) внутри директории. Очень удобно, не нужно получать список и потом склеивать пути.

item.is_dir() — проверяет, является ли объект папкой. На основе этого мы рисуем разные эмодзи (📁 для папок, 📄 для файлов). Это мелочь, но интерфейс становится нагляднее.

Почему pathlib, а не os.listdir()? pathlib делает код короче и читаемее. Не нужно использовать os.path.join() для создания полного пути. Сравните: os.path.join(os.getcwd(), name) против Path.cwd() / name. Второй вариант интуитивно понятен даже новичку.

Функция delete_folder() — удаление с проверками
python
def delete_folder():
    name = input("Введите имя папки для удаления: ")
    target = Path.cwd() / name

    if not target.exists():
        print(f"Папка '{name}' не найдена")
        return

    try:
        target.rmdir()  # удаляет только пустую папку
        print(f"Папка '{name}' удалена")
    except OSError:
        print(f"Папка '{name}' не пуста. Удалите сначала всё внутри.")
Валидация: Программа не бросается сразу удалять. Сначала проверяет, существует ли объект (target.exists()). Это спасает от ошибки FileNotFoundError.

Безопасность: target.rmdir() удаляет только пустые папки. Если внутри есть файлы или другие папки, возникнет исключение OSError, и программа вежливо сообщит: «Папка не пуста». Это сделано намеренно, чтобы случайно не стереть важные данные.

Обработка ошибок: Блок try...except ловит ошибку и превращает её в понятное сообщение для пользователя. Программа не падает, а продолжает работать.

5. Функция open_file_manager() — магия os.system()
python
import os

def open_file_manager():
    os.system(FILE_MANAGER_CMD)
    print("Файловый менеджер открыт")
Это, пожалуй, самая короткая, но и самая интересная функция. os.system() позволяет выполнить любую команду так, как если бы вы ввели её в терминале (или командной строке Windows). Когда мы передаём туда "explorer .", операционная система запускает свой стандартный файловый менеджер и открывает в нём текущую папку (. означает «текущая директория»).

Простота и мощь: Всего одна строка кода — и у нас появляется графическое окно с файлами. Это наглядно показывает, как Python может быть «клеем» для других программ.

Ограничения: os.system() — это очень простая, «топорная» функция. Она не даёт никакого контроля над запущенным процессом. Мы не можем получить вывод команды, проверить, успешно ли она завершилась, или передать ей данные на ходу. Для более серьёзных задач используют модуль subprocess, но для нашего учебного примера os.system() подходит идеально.

6. Главный цикл и «режим киоска»
Функция main() — это то, что делает программу удобной для пользователя:

python
def main():
    while True:
        print_menu()
        choice = get_user_choice()

        if choice == 0:
            print("До свидания!")
            break
        # ... вызов функций для choice 1-6 ...
        else:
            print("Неверный выбор. Попробуйте снова.")

        input("\nНажмите Enter, чтобы продолжить...")
        os.system(CLEAR_CMD)  # очищаем экран
Бесконечный цикл (while True): Программа работает, постоянно показывая меню, пока пользователь сам не захочет выйти.

Очистка экрана: После каждого действия вызывается os.system(CLEAR_CMD). Это создаёт эффект «чистого листа»: меню всегда находится вверху экрана, и его не нужно искать среди старых сообщений. Это сильно повышает удобство.

Пауза перед очисткой: input("\nНажмите Enter, чтобы продолжить...") делает так, что пользователь успевает прочитать результат выполнения команды. Без этой паузы экран очистился бы мгновенно, и вы бы ничего не увидели.

7. Итог: чему конкретно учит эта программа
Кросс-платформенность: Программа — отличный пример того, как писать код, который работает на Windows, macOS и Linux.

Современная работа с путями: Модуль pathlib — это стандарт де-факто в современном Python, и на этом примере видно, почему он так хорош.

Взаимодействие с ОС: Библиотеки os и sys — это главные «ворота» в операционную систему из Python. Программа показывает, как через них запускать команды, проверять платформу и управлять файлами.

Обработка ошибок и валидация: Простые, но эффективные проверки (if not target.exists():) и блоки try...except делают программу устойчивой к ошибкам пользователя.

Проектирование интерфейса: Разбивка на меню, очистка экрана, пауза для чтения результата — эти приёмы превращают скучный скрипт в полноценное приложение, с которым приятно работать.

Этот файловый менеджер — не просто домашняя работа, а минимальный, но работающий шаблон для множества других консольных утилит: от бэкаперов до простых инсталляторов.
