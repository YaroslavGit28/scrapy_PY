
### Обзор обучающей программы на Scrapy — подробно и «по‑человечески»

Эта документация написана так, чтобы в ней разобрался человек, который **только начинает** знакомство с веб-парсингом и фреймворком Scrapy.
---

#### 1. Что вообще делает этот проект

Это **демонстрационный стенд** для библиотеки Scrapy. Scrapy — это мощный фреймворк для извлечения данных с веб-сайтов (парсинга). Но у него есть проблема: новичков пугает его структура. Там нужно создавать проекты, разбираться в куче файлов (`items.py`, `pipelines.py`, `settings.py`, `middlewares.py`), писать пауков в определённом стиле. Это похоже на изучение реактивного самолёта, когда тебе нужно просто слетать в соседний город.

Мой проект — это **разобранный на кирпичики** Scrapy. Я взял каждую важную концепцию и вынес её в отдельный, максимально простой скрипт. Вот что внутри:

1.  **`demo_01_basics.py`** — что такое Request и Response (база, на которой всё держится)
2.  **`demo_02_selectors.py`** — как искать данные в HTML через CSS и XPath
3.  **`demo_03_spider_flow.py`** — как паук ходит по страницам и переходит по ссылкам
4.  **`demo_04_items_pipelines.py`** — как структурировать данные и чистить их перед сохранением
5.  **`demo_05_settings_export.py`** — как настраивать паука и сохранять результат в файл

И есть **`run_all.py`** — запускалка, которая прогоняет всё по очереди. Вы просто пишете `python src/run_all.py` и смотрите, как работает каждая часть Scrapy, без необходимости создавать полноценный проект.

---

#### 2. Почему это важно: проблема, которую решает проект

Scrapy — это «тяжёлая артиллерия» парсинга. Он умеет:
- Делать сотни запросов в секунду асинхронно
- Автоматически следовать по ссылкам (обходить весь сайт)
- Обрабатывать данные через цепочку пайплайнов
- Экспортировать в JSON, CSV, XML и куда угодно

Но официальная документация и туториалы часто предлагают сразу создавать полноценный проект командой `scrapy startproject`. В результате новичок видит десяток файлов и падает духом. «Где тут писать мой код? Что за классы? Зачем столько всего?»

Этот проект решает проблему **«с чего начать»**. Я вырезал весь boilerplate-код и оставил только суть. Каждый скрипт можно запустить и сразу увидеть результат. А потом, когда станет понятно, из каких кирпичиков состоит дом, можно переходить к строительству полноценного проекта.

#### 3. Демо №1: `demo_01_basics.py` — что такое Request и Response

Самый первый и важный скрипт. Он показывает два главных объекта Scrapy.

```python
from scrapy.http import Request, HtmlResponse

# Request — это то, что мы отправляем на сервер
req = Request("https://example.com", method="GET")
print(f"Request URL: {req.url}, метод: {req.method}")

# Response — это то, что мы получаем от сервера
html = "<html><body><h1>Привет, мир!</h1></body></html>"
resp = HtmlResponse(url="https://example.com", body=html, encoding='utf-8')

print(f"Response статус: {resp.status}")
print(f"Заголовок: {resp.css('h1::text').get()}")
```

**Что здесь происходит:**

- `Request` — это заявка на получение страницы. В ней есть URL, метод (GET, POST), заголовки, куки, тело запроса.
- `Response` — это ответ сервера. В нём есть HTML-код (или JSON, или что угодно), статус (200, 404, 500), заголовки.

**Зачем это нужно:** Вся работа Scrapy крутится вокруг этих двух объектов. Паук (spider) возвращает `Request`'ы, а получает `Response`'ы. Понимание этой пары — 50% успеха.

#### 4. Демо №2: `demo_02_selectors.py` — как искать элементы

Это демо показывает, как вытаскивать данные из HTML. Scrapy поддерживает два языка запросов: CSS-селекторы и XPath.

```python
html = """
<div class="products">
    <div class="item" data-id="1">Товар 1</div>
    <div class="item" data-id="2">Товар 2</div>
</div>
"""
response = HtmlResponse(url="test", body=html, encoding='utf-8')

# CSS-селекторы — как в веб-вёрстке
items_css = response.css('div.item')
for item in items_css:
    title = item.css('::text').get()
    data_id = item.css('::attr(data-id)').get()
    print(f"CSS: {title}, id={data_id}")

# XPath — более мощный язык
items_xpath = response.xpath('//div[@class="item"]')
for item in items_xpath:
    title = item.xpath('text()').get()
    print(f"XPath: {title}")
```

**CSS vs XPath:**

| CSS | XPath |
|-----|-------|
| Короче и привычнее для верстальщиков | Мощнее, может идти вверх по дереву (`parent::`) |
| Не умеет искать по тексту | Умеет: `//a[contains(text(), "Купить")]` |
| Не умеет искать по части атрибута | Умеет: `//div[starts-with(@id, "user-")]` |

#### 5. Демо №3: `demo_03_spider_flow.py` — как паук ходит по сайту

Это сердце Scrapy. Скрипт показывает полный цикл работы паука:

```python
import scrapy
from scrapy.crawler import CrawlerProcess

class SimpleSpider(scrapy.Spider):
    name = "demo_spider"
    
    def start_requests(self):
        # Шаг 1: с чего начинаем
        print("Начинаем с главной страницы")
        yield scrapy.Request("https://example.com", callback=self.parse)
    
    def parse(self, response):
        # Шаг 2: парсим главную страницу
        print(f"Парсим: {response.url}")
        
        # Находим все ссылки
        for link in response.css('a::attr(href)').getall()[:3]:  # первые 3 ссылки
            print(f"  Нашли ссылку: {link}")
            # Шаг 3: переходим по каждой ссылке
            yield response.follow(link, callback=self.parse_page)
    
    def parse_page(self, response):
        # Шаг 4: парсим страницу, на которую перешли
        title = response.css('title::text').get()
        print(f"  Страница {response.url}: заголовок = {title}")
        yield {"url": response.url, "title": title}
```

**Что здесь важно:**

- `start_requests` — точка входа. Можно вернуть несколько начальных URL.
- `parse` — обработчик ответа. Здесь мы извлекаем данные и решаем, куда идти дальше.
- `response.follow` — магический метод. Он сам понимает, полный это URL или относительный, и не надо мучиться с `urljoin`.

**Почему это круто:** Scrapy сам управляет асинхронными запросами. Вы просто говорите «перейди по этой ссылке и вызови такой-то метод», а он сам всё распараллеливает, ждёт ответы, не блокирует выполнение.

#### 6. Демо №4: `demo_04_items_pipelines.py` — структура и чистка данных

Это демо показывает, как организовать данные, чтобы не было хаоса из словарей.

**Item — это схема данных:**

```python
from scrapy import Item, Field

class ProductItem(Item):
    title = Field()      # название товара
    price = Field()      # цена
    url = Field()        # ссылка
```

**Pipeline — это обработчик:**

```python
class CleanPricePipeline:
    def process_item(self, item, spider):
        if item.get('price'):
            # Убираем знак доллара, лишние пробелы
            item['price'] = item['price'].replace('$', '').strip()
        if item.get('title'):
            # Обрезаем пробелы по краям
            item['title'] = item['title'].strip()
        return item
```

**Как это работает вместе:**

1. Паук создаёт `ProductItem`, заполняет поля
2. Scrapy передаёт этот Item во все Pipeline'ы по очереди
3. Каждый Pipeline может изменить Item, отфильтровать (вернуть `None`) или выбросить исключение
4. В конце Item попадает в экспортёр (JSON, CSV, база данных)

**Зачем это нужно:** В реальном проекте у вас будет не один паук, а десяток. Items и Pipelines помогают держать логику обработки данных в одном месте, а не размазывать её по всем паукам.

#### 7. Демо №5: `demo_05_settings_export.py` — настройки и сохранение

Последнее демо показывает, как управлять поведением паука и куда сохранять результат.

```python
settings = {
    "USER_AGENT": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36",
    "DOWNLOAD_DELAY": 1,  # ждём 1 секунду между запросами (вежливый парсинг)
    "FEEDS": {
        "output.json": {"format": "json", "encoding": "utf-8"},
        "output.csv": {"format": "csv"},
    }
}

process = CrawlerProcess(settings)
process.crawl(MySpider)
process.start()
```

**Что можно настраивать:**

| Настройка | Что делает |
|-----------|------------|
| `USER_AGENT` | Представляемся браузером, чтобы сайты нас не блокировали |
| `DOWNLOAD_DELAY` | Задержка между запросами к одному домену (не долбим сервер) |
| `CONCURRENT_REQUESTS` | Сколько запросов одновременно (по умолчанию 16) |
| `RETRY_TIMES` | Сколько раз повторять при ошибке |
| `FEEDS` | Куда и в каком формате сохранять результат |

**Почему это важно:** Без `DOWNLOAD_DELAY` вы рискуете положить небольшой сайт. Без `USER_AGENT` вас быстро забанят. Без `FEEDS` результат останется только в консоли.

#### 9. Итог: чему конкретно учит этот проект

1. **Scrapy — это не страшно.** Он большой, но он разбирается на понятные куски.
2. **Request и Response — основа всего.** Не надо их бояться.
3. **CSS и XPath — инструменты поиска.** CSS для простых случаев, XPath для сложных.
4. **Паук — это автомат по обходу сайтов.** Он стартует, парсит, переходит по ссылкам, парсит дальше.
5. **Items и Pipelines — порядок в данных.** Структурируй и чисти в одном месте.
6. **Настройки — это вежливость и безопасность.** Задержки, user-agent, экспорт.

По сути, этот проект — это **мостик** между «я ничего не понимаю в Scrapy» и «я могу написать своего первого паука». Он не заменяет официальную документацию, но делает первый шаг гораздо менее болезненным.
